MailWizz
The Dumb Delivery Pipe
Self-hosted PHP email marketing platform. It sends emails via SMTP and reports engagement back via webhooks. Zero intelligence — pure plumbing.

✓ PROS

  • Self-hosted — full control, no vendor lock-in
  • Handles SMTP relay, bounce processing, throttling
  • Free (open source + one-time license)
  • Has a REST API (lists, subscribers, campaigns)
  • Built-in bounce/complaint handling
  • Campaign templates and scheduling
  • Already set up and working

✗ CONS

  • PHP — hard to customize, slow development
  • Limited API (clunky pagination, no batch ops)
  • No AI/ML integration capability
  • Clunky admin UI
  • Community shrinking
  • No real-time event streaming
  • Webhook format is non-standard
  • Can't handle advanced segmentation logic

The Intelligence Engine
The Self-Healing Loop
Step by step, with exact API calls, request bodies, and which file makes each call.
Step 1 — Pick Contacts
MarketX → Refinery

MarketWriter's workflow engine queries Refinery to build the perfect audience based on ICP filters.

// MarketX: workflow-execution-processor.ts → refinery-client.ts POST /api/v1/contacts/search Authorization: Bearer rnx_live_xxx { "filters": { "primary_industry": "SaaS", "seniority_level": ["VP", "Director", "C-Level"], "company_employee_count": { "gt": "50" }, "_verification_status": { "not": "bounced" } }, "per_page": 500, "sort_by": "_ingested_at" } // Response: { "data": [{ up_id, business_email, company_name, ... }], "meta": { total: 12450 } }
Step 2 — Verify Emails
MarketX → Refinery

Before sending, check verification status. Remove bounced/risky emails.

POST /api/v1/verify/bulk-status Authorization: Bearer rnx_live_xxx { "emails": ["ceo@acme.com", "vp@startup.io", ...] } // Response: { "data": [ { "email": "ceo@acme.com", "found": true, "verification_status": "safe" }, { "email": "vp@startup.io", "found": true, "verification_status": "rejected" } ], "meta": { "requested": 500, "found": 487 } }
Step 3 — Send via MTA
MarketX → MailWizz

MarketWriter pushes verified contacts through the MTA Adapter to MailWizz. MailWizz fires via SMTP.

// MarketX: MailWizzAdapter.ts (via MTAAdapter interface) mtaAdapter.createList("Campaign-2026-03-19") mtaAdapter.addSubscribers(listId, verifiedContacts) mtaAdapter.sendCampaign({ listId, subject, htmlBody, fromEmail })
Step 4 — Webhooks Flow Back
MailWizz → Refinery

MailWizz POSTs engagement events to Refinery. Stored in ClickHouse engagement_events table.

// MailWizz → Refinery (Phase 2) POST /api/v1/webhooks/bounce { email, bounce_type, campaign_uid } POST /api/v1/webhooks/open { email, campaign_uid, ip, timestamp } POST /api/v1/webhooks/click { email, campaign_uid, url, timestamp } POST /api/v1/webhooks/reply { email, campaign_uid, timestamp }
Step 5 — Brain Reads Stats
MarketX ← Refinery

MarketWriter queries aggregated intelligence to measure what worked and what didn't.

// MarketX: self-healing.ts → refinery-client.ts (Phase 4) GET /api/v1/stats/deliverability // Response: { bounce_rate, inbox_rate, complaint_rate, by_domain: {...} } GET /api/v1/stats/engagement // Response: { open_rate, click_rate, reply_rate, by_segment: {...} }
Step 6 — Adjust & Repeat
MarketX internal

AI updates ICP scores, timing windows, content strategy. Mastery agents learn from results. The loop restarts, smarter.

// MarketX internal: mastery agents + marketing coach // Files: marketing-coach-processor.ts, self-healing.ts // Tables: mastery_agent_configs, brain_decisions, contact_decisions // → Updates ICP weights, sending time preferences, subject line patterns // → Next iteration uses refined filters in Step 1
